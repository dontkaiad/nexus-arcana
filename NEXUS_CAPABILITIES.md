## КАК РАБОТАЕТ NEXUS В ЦЕЛОМ

Пользователь пишет текст в Telegram на русском языке (поддерживается раскладка EN→RU и опечатки).
Текст проходит pipeline:
1. `maybe_convert()` — конвертер раскладки (pfgjvyb → запомни)
2. `original_text` сохраняется ДО spell correction
3. Spell correction через Claude Haiku (исправляет опечатки, не меняет смысл)
4. `classify(original_text)` — pre-filters + routing
5. `process_item()` → нужный handler
6. Ответ пользователю

**Мультиюзер:** каждый запрос фильтруется по TG ID → находит запись в базе Пользователи → читает права (checkboxes) и tz_offset. Если права отсутствуют — отказ.

**Часовой пояс:** у каждого пользователя свой tz_offset в базе Пользователи. Все даты/напоминания рассчитываются с учётом его timezone. Пользователь может обновить timezone написав "я в москве" / "живу в берлине" / "у меня сейчас 15:00" — бот определяет UTC offset, обновляет базу Пользователи И сохраняет факт в Память (🏠 Быт).

## ✅ ЗАДАЧИ

### Создание
Триггеры (pre-filter `_BUY_TASK_RE` + Claude):
- "купить корм коту" → задача категории 🐾 Коты
- "написать Маше завтра в 15" → задача с дедлайном, категория 👥 Люди
- "позвонить врачу в пятницу напомни за час" → дедлайн + напоминание
- "погладить кота каждый день в 9 утра" → повторяющаяся задача

Если дедлайн/напоминание не указаны → один объединённый вопрос:
```
📌 купить корм
🗂 Коты · Средний
📅 Дедлайн: не указан
🔔 Напоминание: нет

❓ Уточни:
— Когда сделать?
— Напомнить?

[✅ Сохранить] [❌ Отмена]
```

### Категории задач
`🐾 Коты · 🏠 Жилье · 🚬 Привычки · 🍜 Продукты · 🍱 Кафе/Доставка · 🚕 Транспорт · 💅 Бьюти · 👗 Гардероб · 💻 Подписки · 🏥 Здоровье · 🕯️ Расходники · 📚 Хобби/Учеба · 💳 Прочее · 👥 Люди`

Haiku определяет категорию автоматически. Если категория не найдена в Notion → бот предлагает добавить новую с подтверждением [✅ Добавить] [📋 Выбрать из существующих] [💾 Без тега].

### Редактирование задачи
После создания задачи можно написать:
- "измени категорию на коты" → `match_select()` нормализует → обновляет в Notion
- "перенеси на завтра" → обновляет дедлайн
- Если значение не найдено в Notion → "❓ Не нашла «одежда». Похожие: 👗 Гардероб..."

### Выполнение задачи
- "сделала купить корм" / "готово" / "выполнено"
- Кнопка [✅ Сделано!] в уведомлении о напоминании
- Кнопка [✅ Выполнено!] в уведомлении о дедлайне
- При нажатии → статус Done в Notion + похвала `get_praise("task_done")`

### Кнопки в напоминаниях
Напоминание: [✅ Сделано!] [❌ Не сделал]
Дедлайн: [✅ Выполнено!] [⏳ Отложить]
При "Не сделал" / "Отложить" → предлагает перенести на +1 / +3 дня

### Повторяющиеся задачи
Поля: Повтор (Нет/Ежедневно/Еженедельно/Ежемесячно) + День недели + Время повтора
При выполнении повторяющейся задачи:
- Дедлайн сдвигается на следующий цикл
- Старый APScheduler job удаляется, создаётся новый
- Статус → Not started

`restore_reminders_on_startup()` при старте бота:
- Проход 1: Напоминание > now → создаёт job
- Проход 2: Повторяющиеся с истёкшим напоминанием → `_next_cycle_date()` → обновляет Notion → job

### Список задач
`/tasks` → активные задачи (Not started / In progress), пагинация по 10

### Auto-suggest в память
После создания задачи → `auto_suggest_memory()`:
Haiku анализирует текст → если есть запоминаемый факт:
"💡 Сохранить в память: royal canin indoor 2кг (Коты)?" [🧠 Да] [✗ Нет]

## 💰 ФИНАНСЫ

### Запись расхода
По умолчанию расход — если нет маркеров дохода ("получила", "заработала", "зарплата", "вернули"):
- "450р такси" → -450₽ · Транспорт · Карта
- "кафе 1200" → -1200₽ · Кафе/Доставка
- "пицца 800р наличные" → с указанием источника

### Запись дохода
Только при явных маркерах:
- "получила зарплату 50000" → +50000₽ · Зарплата
- "вернули долг 3000" → +3000₽

### Категории финансов
`🐾 Коты · 🏠 Жилье · 🚬 Привычки · 🍜 Продукты · 🍱 Кафе/Доставка · 🚕 Транспорт · 💅 Бьюти · 👗 Гардероб · 💻 Подписки · 🏥 Здоровье · 🕯️ Расходники · 📚 Хобби/Учеба · 💰 Зарплата · 🔮 Практика · 💳 Прочее`

Источники: `💳 Карта · 💵 Наличные · 🔄 Бартер`

### Бюджетные лимиты
Хранятся в Памяти (категория 💰 Лимит):
- "лимит на сигареты 3000р в месяц" → сохраняет: "лимит: 🚬 Привычки — 3000₽/мес"
- "измени лимит на кафе 7000р" → обновляет существующую запись

`_check_budget_limit()` вызывается после КАЖДОГО расхода:
- Считает ВСЕ расходы с 1-го числа текущего месяца
- < 80% → тихо
- 80-100% → "⚠️ Уже потрачено 2500₽ из 3000₽ на 🚬 Привычки"
- > 100% → "🚨 Превышен лимит на 🚬 Привычки: 3200₽ из 3000₽"

### Статистика
`/finance_stats` → сводка за текущий месяц:
- Общие расходы/доходы/баланс
- По категориям с лимитами: 🟢 (60%) / 🟡 (85%) / 🔴 (+200₽)
- Ревью по лимитам:
  - < 50% → 🎉 похвала + `get_praise("finance_under_limit")`
  - 50-80% → без комментария
  - 80-100% → ⚠️
  - > 100% → 😬 совет на следующий месяц

Phase 10 — мультимесячная аналитика:
- "за N месяцев" → разбивка по месяцам + среднее
- "сравни месяцы" → текущий vs предыдущий ↑↓
- Прогноз до конца месяца если день 5–24 и темп высокий

`/finance` → список транзакций с пагинацией
"сколько потратила на кафе?" → статистика по категории + ревью если есть лимит

## 💡 ЗАМЕТКИ

### Создание
- "заметка: посмотреть сериал Severance"
- "идея: витрина для кристаллов"
- "запомни рецепт: 2 яйца..."

Если теги не найдены в Notion → предлагает добавить новые:

[✅ Добавить] [📋 Выбрать из существующих] [💾 Без тегов]

### Редактирование
- "измени тег на идея" → обновляет тег заметки
- Fuzzy-матчинг: "одежда" → ищет похожее среди существующих тегов

### Список и поиск
`/notes` → все заметки с пагинацией по 10

### Дайджест
`/notes_digest` → ручной вызов
Cron: каждое воскресенье 07:00 UTC (12:00 UTC+5):
- Заметки где Дата < 7 дней назад, макс 5 штук
- Формат: "💡 У тебя 5 заметок без действий: ..."
- После дайджеста: "удали все заметки про расходники" → удаляет из _last_digest_results

## 🧠 ПАМЯТЬ

### Сохранение
Pre-filter `_MEMORY_SAVE_RE` (до spell correction):
- "запомни что маша не ест мясо" → 🧠 Запомнила [👥 Люди]: маша не ест мясо
- "маша это моя подруга" → авто-распознавание без "запомни"
- "у меня аллергия на арахис" → категория 🏥 Здоровье
- "алуна не ест курицу" → категория 🐾 Коты, Связь=алуна
- "лимит на кафе 5000р" → категория 💰 Лимит

Haiku парсит: факт, категорию, связь (имя), ключ (snake_case).
При сохранении лимита — проверяет существующую запись по ключу → обновляет вместо создания дубля.

### Категории памяти (11)
`🧠 СДВГ · 👥 Люди · 🏥 Здоровье · 🛒 Предпочтения · 💼 Работа · 🏠 Быт · 🔄 Паттерн · 💡 Инсайт · 🔮 Практика · 🐾 Коты · 💰 Лимит`

### Поиск
Pre-filter `_MEMORY_SEARCH_RE`:
- "напомни про машу"
- "что ты знаешь о батоне"
- "расскажи про корм"

`asyncio.gather` — параллельные запросы:
1. NOTION_DB_MEMORY: Связь contains + Текст contains
2. NOTION_DB_FINANCE: Описание contains
3. NOTION_DB_TASKS: Задача contains (статус ≠ Done)

Нормализация падежей: "машу"→"маш"→finds "маша", "батона"→"батон"

Формат вывода:
```
🧠 Память (найдено 2):
👥 маша не ест мясо · Люди · 2026-03-22
👅 маша это подруга · Люди · 2026-03-22

💰 Финансы:
450₽ такси маша · 2026-03-22

✅ Задачи:
написать Маше · до 2026-03-23
```

Прямо в результатах — чекбоксы:
- [☐ запись] → нажать = деактивировать (Актуально=false, история остаётся)
- [↩️ Восстановить] — если запись деактивирована
- [🗑️ Удалить все (N)] → реальное удаление

### Удаление
- "удали из памяти про машу мясо" → показывает список с чекбоксами
- toggle = реальное удаление (архивирование в Notion)
- [🗑️ Удалить все] → удаляет все найденные

### Деактивация vs Удаление
- Деактивация (через чекбокс в поиске): Актуально=false, запись остаётся, можно восстановить
- Удаление (команда "удали из памяти"): архивирование, навсегда

### Auto-suggest из задач и финансов
После создания задачи или записи расхода → Haiku анализирует текст:
- Если есть запоминаемый факт → "💡 Сохранить в память: [факт]?" [🧠 Да] [✗ Нет]
- При согласии → сохраняет в NOTION_DB_MEMORY

### Связь с timezone
"я в спб" → обновляет tz_offset в Пользователях И сохраняет в Память: "в Санкт-Петербурге" (🏠 Быт)

## 🧠 СДВГ-ФИЧИ

### /adhd — персональный СДВГ-профиль
Sonnet генерирует профиль на основе всех фактов категории 🧠 СДВГ.
Группировка: Паттерны / Стратегии / Триггеры / Особенности.

### Совет при сохранении СДВГ-факта
При добавлении факта в категорию 🧠 СДВГ → Sonnet генерирует персональный совет (женский род, имя Кай).

### Еженедельный СДВГ-дайджест
Cron: каждое воскресенье 08:00 UTC (13:00 UTC+5).
Sonnet анализирует СДВГ-факты + задачи за неделю → инсайты и рекомендации.

### Нудж при прокрастинации
`_check_procrastination_nudge()` — Haiku анализирует задачи со статусом Not started и истёкшими дедлайнами → мягкий нудж.

## ⚙️ СИСТЕМА

### Похвала `get_praise(context)`
Контексты и когда вызывается:
- `"task_done"` → при нажатии ✅ Сделано
- `"memory_saved"` → при сохранении факта в память
- `"finance_under_limit"` → в /finance_stats если потрачено < 50% лимита
- `"generic"` → fallback

### Новые опции Select
Если Haiku предложил категорию/тег которого нет в Notion:
"💡 Не нашёл тег(и): #Новый. Существующие: [список]"
[✅ Добавить #Новый] [📋 Выбрать из существующих] [💾 Без тегов]
НИКОГДА не добавляет автоматически без подтверждения.

### match_select()
При редактировании категории/тега — нормализует введённое пользователем:
"одежда" → ищет в существующих опциях → "👗 Гардероб"
Если не найдено → "❓ Не нашла «одежда». Похожие: 👗 Гардероб, 💳 Прочее — уточни?"
НЕ пишет в Notion без совпадения.

### Пагинация
Для всех списков > 10 элементов через core/pagination.py.
Кнопки [← Назад] [→ Далее] под списком.

### Порядок pre-filters в classify()
```
1. _DEACTIVATE_RE     → "неактуально [N]"
2. _MEMORY_SAVE_RE    → "запомни что..."
3. _MEMORY_SEARCH_RE  → "напомни про..."
4. _MEMORY_DELETE_RE  → "удали из памяти..."
5. _EDIT_NOTE_RE      → "измени тег/метку..."
6. _EDIT_RE           → "измени категорию/приоритет..."
7. _DONE_RE           → "сделала/готово/выполнено..."
8. _BUY_TASK_RE       → "купить X" без суммы → задача
9. _TZ_RE             → "я в X / живу в X / utc+N"
10. _STATS_RE         → "сколько потратила на X"
11. Claude            → всё остальное
```

### Команды
```
/tasks          → список активных задач
/notes          → список заметок
/notes_digest   → дайджест старых заметок (ручной)
/finance        → история финансов
/finance_stats  → статистика за месяц + ревью лимитов
/memory         → список памяти (с фильтром: /memory коты)
/adhd           → СДВГ-профиль
/help           → справка
```
