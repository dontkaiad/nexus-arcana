# E2E тесты (tests/e2e/, Telethon, живые боты)

Не запускаются pytest'ом — это отдельные раннеры против реальных ботов.
Юнит/интеграционные тесты живут в `tests/` и `tests/integration/` (pytest).
Smoke-тесты с реальными API: `python tests/e2e/smoke.py`.

## Первый запуск (одноразовая настройка):

1. Зайди на https://my.telegram.org
2. Войди по номеру → API Development Tools → Create Application
3. Скопируй API ID и API Hash
4. Создай `tests/.env.test` и впиши туда значения
5. `pip install telethon`

## Запуск:

```bash
cd tests/e2e
export $(cat ../.env.test | xargs)
python run_all.py
```

При первом запуске Telethon попросит номер телефона и код — это одноразово,
потом сессия сохраняется в `e2e_session.session`.

## Что тестируется:

### ☀️ Nexus (30+ тестов):
- Все команды (/start, /help, /today, /tasks, /stats, /finance, /budget, /list, /memory, /adhd, /notes, /tz)
- Задачи: создание, дедлайн, повтор, интервал, закрытие, отмена
- Финансы: расход, доход, лимит
- Списки: покупки, инвентарь, поиск
- Заметки и память: создание, поиск, удаление
- Edge cases: EN раскладка, пробелы, длинные сообщения

### 🌒 Arcana (35+ тестов):
- Все команды (/start, /help, /stats, /finance, /list, /grimoire, /tz)
- CRM: создание клиента, find-or-create, досье, долги
- Расклады: текстом (простой, с клиентом, с оплатой), правка, сохранение
- Ритуалы: личные, клиентские с оплатой
- Работы: создание, закрытие
- Статистика: верификация, /stats
- Память: создание, поиск, удаление
- Гримуар: запись, просмотр
- Списки: расходники, инвентарь
- Финансы практики
- Edge cases
