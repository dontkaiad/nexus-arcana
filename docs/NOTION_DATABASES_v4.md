## Расхождения схема ↔ код (warnings)

Найдено при автоматическом сравнении схем Notion с константами в коде
(`LIST_CATEGORIES`, `MEM_CATEGORIES`, finance cats в classifier.build_system).

### 1. 🗒️ Списки.Категория — есть в Notion, нет в `LIST_CATEGORIES`

- ⚠️ **`🔄 Бартер`** — используется Arcana barter-флоу, но не объявлен
  в `core/list_manager.py:LIST_CATEGORIES`. Haiku-парсер не предлагает
  её в категориях. Не критично пока Arcana пишет напрямую через
  match_select, но рассинхрон.

### 2. 💰 Финансы.Категория — есть в Notion, нет в classifier promp'те

- ⚠️ **`🎲 Импульсивные`**
- ⚠️ **`🔮 Практика`**
- ⚠️ **`🕯️ Расходники`**

Эти категории видимо созданы Кай вручную в Notion. Classifier
(`core/classifier.py:build_system`) их не знает, поэтому Haiku при
парсинге расходов не выберет их — отправит в `💳 Прочее` либо галлюцинирует.

### 3. 💰 Финансы.Категория — есть в `build_system`, нет в Notion

- ⚠️ **`🤖 Боты`** — упомянут в classifier как категория для разработки
  ботов, но опции в Notion НЕТ. При write Notion тихо создаст её
  через `match_select` fallback (либо `💳 Прочее`).

## Технические правила

- **Status-поля** (`Статус`) пишем через `_status()`, не `_select()` — иначе Notion 400.
- **Select-поля с эмодзи-префиксом** — обязательно через `match_select(db_id, prop, value)` чтобы попасть в реальную опцию.
- **Финансы / Память / Списки** общие для двух ботов; различаем по полю `Бот` (`☀️ Nexus` / `🌒 Arcana`).
- **Расклады / Ритуалы** пишут финансовые поля прямо в свою запись (`Сумма`, `Оплачено`, `Источник`). В 💰 Финансы НЕ дублируется — касса P&L агрегирует через `core/cash_register.py`.
- **Дно колоды** (🃏 Расклады) — `rich_text`, не `select`. Запись через `_text(card_name)`.
- **Trailing space в названии поля** — `🔮 Работы ` (с пробелом) в схеме 🗒️ Списки. Поддерживаем оба варианта при чтении.
- **`data.get("key") or default`** — НЕ `data.get("key", default)` (второй вернёт `None` для `null` Notion).
- **emoji-канонизация** для select опций (`☀️ Nexus`, не `Nexus`; `🐾 Коты`, не `Коты`).
