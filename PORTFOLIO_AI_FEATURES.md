# PORTFOLIO_AI_FEATURES — Prompt Engineering & AI Architecture

Реальные примеры prompt engineering и AI-архитектурных решений из кодовой базы
двойной системы Telegram-ботов (☀️ Nexus — личный ассистент, 🌒 Arcana — CRM
практики). Каждая фича: **где** (файл:строка), **что делает**, **почему это
интересно** с точки зрения prompt engineering.

> Все примеры — production-код. Личные данные (профиль пользователя, имена
> клиентов, суммы) в этом документе обобщены/заменены плейсхолдерами; в коде
> они тянутся из Notion в рантайме.

## 1. Cost-Aware Model Routing — центральное архитектурное решение

Пользователь платит за каждый токен из своего кармана, поэтому выбор модели —
не деталь, а архитектурный инвариант, защищённый тестом.

### 1.1 Дефолт — Haiku, Sonnet только явно
- **`core/config.py:65-66`** — единственный источник model-id:
  ```python
  MODEL_HAIKU  = "claude-haiku-4-5-20251001"
  MODEL_SONNET = "claude-sonnet-4-6"
  ```
- **`core/claude_client.py:32`** — `used_model = model or config.model_haiku`.
  Любой `ask_claude()` без явного `model=` уходит на Haiku.
- **Почему круто:** дешёвая модель — это *поведение по умолчанию*, а не то, что
  надо помнить. Дорогая модель требует осознанного `model=...`. Это «pit of
  success» на уровне API-обёртки.

### 1.2 Тест-страж против ценовой регрессии
- **`tests/test_models_audit.py`** — чисто статический аудит (грепает исходники,
  никаких живых вызовов):
  - `HAIKU_REQUIRED` (13 файлов: router, deleter, reply_update, finance, stats,
    works, clients, grimoire, rituals, notes, notes_smart_select, nexus_bot,
    miniapp/today) — каждый ОБЯЗАН содержать `model="claude-haiku...`. Иначе тест
    падает.
  - `SONNET_LEGIT` (4 файла: `core/memory.py`, `core/vision.py`,
    `arcana/handlers/sessions.py`, `miniapp/.../arcana_sessions.py`) — whitelist
    мест, где Sonnet оправдан. Sonnet где угодно ещё = красный тест.
- **Почему круто:** это **eval ценовой архитектуры**. Cost-control обычно живёт в
  ревью и забывается; здесь он формализован как регрессионный тест. Новый
  контрибьютор не может случайно «уронить» рутинный парсер на Sonnet.

### 1.3 Где Sonnet оправдан (и почему именно там)
| Место | Файл:строка | Почему не Haiku |
|---|---|---|
| Трактовка Таро | `arcana/handlers/sessions.py:254 (TAROT_SYSTEM)`, вызовы `:744, :974, :1412, :1635` | Нарратив + эмпатия + связывание с историей клиента |
| Бюджетная аналитика | `nexus/handlers/finance.py:2313 (BUDGET_SONNET_SYSTEM)`, вызов `:3196` | Многошаговый алгоритм, перепланирование долгов, два варианта вывода |
| Vision (чеки) | `core/vision.py:22-103` | Понимание изображений |
| Саммари сессии (Mini App) | `miniapp/backend/routes/arcana_sessions.py:481` | Синтез нарратива поверх N триплетов |
| Long-form СДВГ-совет | `core/memory.py:459-484` | Контекстная генерация под профиль |
| Поэтичный tip Арканы | `miniapp/backend/routes/arcana_today.py:119` | Тон/стиль важнее скорости |

> Заметка для честности: в коде сосуществуют два литерала Sonnet —
> `claude-sonnet-4-6` (config, `core/memory.py:484`, `nexus/handlers/memory.py:197`)
> и `claude-sonnet-4-20250514` (вызовы в `arcana/handlers/sessions.py`,
> `base.py:268`, `arcana_sessions.py:481`). Кандидат на консолидацию в одну
> константу.

## 2. Multi-Domain Intent Classification — two-tier

Два независимых классификатора по числу ботов, с общим приёмом: **дешёвый
regex-слой ловит очевидное до LLM**, а LLM получает только неоднозначное.

### 2.1 Nexus: классификатор-мегапромпт
- **`core/classifier.py:88` — `build_system(tz_offset)`** строит большой
  динамический system-prompt (файл ~1169 строк): 13+ типов (expense / income /
  task / note / memory_save / memory_search / stats / list_* / arcana_redirect…),
  десятки few-shot примеров, инжект текущей даты/времени.
- **`core/classifier.py:741`** — главный вызов идёт *без* `model=` → дефолтный
  Haiku, `max_tokens=1024`. Самый частотный вызов системы — и он самый дешёвый.
- **Night-mode логика** прямо в промпте: до 05:00 «завтра» = сегодня — снимает
  класс ошибок у ночного пользователя.
- **`arcana_redirect`** (`:166, :248-249, :298`): если в тексте есть слова из
  `ARCANA_KEYWORDS`, но нет явного типа — отправить в соседний бот. Маршрутизация
  между доменами зашита в классификацию.
- **Pre-regex слой** (десятки паттернов) ловит команды/удаления/таймзону/списки
  до Claude — экономия и детерминизм.
- **Почему круто:** один Haiku-вызов решает «куда это» по 13+ направлениям с
  учётом времени суток; «горячий путь» (regex) бесплатный.

### 2.2 Arcana: ROUTER + дизамбигуация по времени глагола
- **`arcana/handlers/base.py:21` — `ROUTER_SYSTEM`** (Haiku, `max_tokens=10`,
  вызовы `:446, :460`): ~21 интент (`session_done/planned/search`,
  `ritual_done/planned/ambiguous`, `new_client`, `grimoire*`, `memory_*`,
  `verify`, `stats`, `nexus_redirect`…). Ответ — **одно слово**.
- **Грамматический дизамбигуатор** (ядро промпта):
  > «глагол в инфинитиве ("сделать", "провести", "разложить") = ЗАПЛАНИРОВАНО
  > (*_planned). Прошедшее время ("сделала", "провела") = СДЕЛАНО (*_done).
  > НИКОГДА не выбирай *_done для инфинитива.»
- Резервный guard на стороне кода (`_PAST_TENSE_RE`): даже если Haiku вернул
  `ritual_done` на инфинитив — система понижает до preview-флоу, а не пишет факт
  сразу (см. `tests/test_intent_fallback.py:76-89`).
- **Почему круто:** морфология русского (вид/время глагола) превращена в сигнал
  планирования. `max_tokens=10` для single-word — минимальная стоимость
  классификации. Двойная защита: prompt-инструкция + код-guard.

## 3. Few-Shot Prompting — везде, где парсинг

Few-shot — основной приём структурированных парсеров. Примеры:

- **`arcana/handlers/works.py:15-41` — `PARSE_WORK_SYSTEM`**: вход→выход прямо в
  промпте, включая относительные даты («завтра» → `YYYY-MM-DDT18:00`, «в пятницу»
  → ближайшая пятница).
- **`arcana/handlers/sessions.py:117` — `PARSE_SESSION_SYSTEM`**: три формата
  ввода раскладов (одиночный триплет / нумерованная сессия / свободная форма
  «вопрос → карты»), для каждого — пример. `max_tokens=4000` (вызов `:649`).
- **`core/lists_parser.py` (`_PARSE_*`)**: примеры категоризации (энергетики →
  🚬 Привычки, корм → 🐾 Коты, лекарства → 🏥 Здоровье), `max_tokens=2000` под
  длинные списки.
- **`tests/test_lists_parser_tech_category.py`**: few-shot-фикс галлюцинации —
  явные примеры «iPhone/ноутбук = 💻 Техника, не путать с 💻 Подписки» +
  негативная инструкция «не путай разовую покупку с подпиской».
- **`arcana/handlers/sessions.py:242 (SESSION_SEARCH_PARSE_SYSTEM)`**: примеры
  извлечения ключевых слов («расклады про работу» → `["работа"]`).
- **Почему круто:** few-shot применяется хирургически — там, где формат
  неоднозначен, и подкреплён тестами, фиксирующими конкретные кейсы галлюцинаций
  (v1.2.1 tech-категория).

## 4. Chain-of-Thought / структурированное рассуждение

### 4.1 Трактовка Таро — пронумерованные правила + жёсткий формат
- **`arcana/handlers/sessions.py:254` — `TAROT_SYSTEM`** (Sonnet, `max_tokens=2000`):
  - 6 пронумерованных правил: значения СТРОГО из справочника, каждая карта
    «Позиция → Название → значение в этой позиции», связать с прошлыми раскладами,
    краткий вывод 2-3 предложения, «БЕЗ поэзии и воды».
  - Анти-паттерн прямо в промпте: «НЕ приписывай картам Прошлое/Настоящее/Будущее
    — триплет это не таймлайн, а три ракурса сути».
  - **HTML-only output guard**: только `<h3>/<b>/<i>/<p>`, «никакого markdown
    (никаких **, ##, * — никогда), никаких `<div>/<span>/инлайн-стилей`».
- **Почему круто:** CoT здесь — это явная структура рассуждения (карта за картой,
  потом синтез), плюс negative constraints против дефолтных привычек модели
  (markdown, временные позиции, «вода»).

### 4.2 Бюджетный алгоритм — многошаговое рассуждение с состоянием
- **`nexus/handlers/finance.py:2313` — `BUDGET_SONNET_SYSTEM`** (Sonnet,
  `max_tokens=4096`, вызов `:3196`):
  - Пошагово: доход − фиксированное = распределяемое; «один долг за раз»;
    детект «тяжёлого месяца»; СДВГ-ориентированные ограничения распределения.
  - **Two-variant output**: при тяжёлом месяце — вариант A («платить по плану»)
    и вариант B («передоговориться по долгу») с нарративом.
  - Строгая JSON-схема с вложенными объектами (income / fixed / debts / variants /
    limits / goals).
- **Почему круто:** один из немногих *оправданных* Sonnet-кейсов — задача требует
  состояния (перенос остатка), пересортировки долгов и генерации плана действий,
  а не извлечения полей.

## 5. Context Injection / RAG-паттерны

### 5.1 Послойная сборка system-prompt (Tarot)
- **`arcana/handlers/sessions.py:729-735`** — один промпт собирается на лету:
  ```python
  system = TAROT_SYSTEM
  if cards_context:  system += "\n\n--- СПРАВОЧНИК КАРТ ---\n" + cards_context
  if memory_context: system += "\n\n--- ПАМЯТЬ ---\n" + memory_context
  if prev_context:   system += "\n\n--- ПРЕДЫДУЩИЕ РАСКЛАДЫ КЛИЕНТА ---\n" + prev_context
  ```
  4 источника: базовые правила + канонические значения карт колоды + личные факты
  из памяти + последние расклады клиента.
- **Почему круто:** retrieval-augmented generation без векторной БД — релевантный
  контекст (справочник конкретной колоды, история именно этого клиента)
  инжектится секциями. Модель «помнит» клиента и не выдумывает значения карт.

### 5.2 Keyword-RAG память (без эмбеддингов)
- **`core/memory.py`**: `save_memory` (`:489`), `search_memory` (`:610`),
  `get_memories_for_context` (`:991`), `auto_suggest_memory` (`:1044`).
  - Парсер факта `_PARSE_SYSTEM` (`:42`, Haiku, `max_tokens=200`): текст → JSON
    `{fact, category, связь, ключ}` с 17 категориями (включая 🦋 СДВГ).
  - Поиск — нормализация токенов (регистр/диакритика/падежи) + поиск по
    Текст/Ключ/Связь + **alias-resolution** (распознаёт «также известна как…»,
    рекурсивно, глубина ≤3).
  - `get_memories_for_context` дедуплицирует по page_id и фильтрует по ярлыку бота
    (Nexus vs Arcana) — общий стор, разный контекст.
- **Почему круто:** прагматичный RAG для маленького персонального корпуса —
  никаких эмбеддингов/инфраструктуры, но есть канонизация алиасов и инжект в
  downstream-промпты.

### 5.3 Auto-suggest памяти по повторению
- **`arcana/handlers/memory.py` / `core/memory.py`**: счётчик на пару
  (intent, тема); на 3-м повторении одного факта → «🧠 Запомнить?». Срабатывает
  только для `session_done / client_info / ritual_done`
  (`tests/test_arcana_memory.py:74-104`).
- **Почему круто:** проактивная память на основе поведенческого сигнала
  (повторение = паттерн), без явной команды пользователя.

## 6. Vision — три отдельных парсера под три задачи

| Задача | Файл:строка | Модель | Особенность промпта |
|---|---|---|---|
| Чеки/банк-операции | `core/vision.py:22 (_RECEIPT_SYSTEM)`, парс `:71-103` | Sonnet, `max_tokens=2048` | Whitelist категорий; доход vs расход по знаку; `math.ceil(abs(amt))` — консервативное округление вверх; неизвестная категория → 💳 Прочее + `need_clarify` |
| Фото расклада Таро | `arcana/handlers/sessions.py:287 (VISION_SYSTEM)`, вызов `:1483` | Vision | Порядок карт «слева-направо, сверху-вниз»; эвристика «лишняя 4-я карта в триплете = дно колоды → в `bottom_card`» |
| Скриншот TG-профиля клиента | `arcana/handlers/clients.py (VISION_CONTACT)` | Vision | Извлечь имя/username/ДР/контакты → JSON, всё `или null` |
- **Почему круто:** не «один vision-промпт на всё», а три специализированных со
  встроенными доменными эвристиками (округление сумм вверх, геометрия раскладки,
  null-safety против галлюцинаций).

## 7. Whisper — голос с graceful degradation

- **`core/voice.py:15` — `transcribe(file_bytes, ...) -> str | None`**:
  OpenAI `whisper-1`, `language="ru"` (`:28-29`). Если нет `OPENAI_API_KEY` —
  возвращает `None` (`:22`), без падения. Вызывается из `nexus/nexus_bot.py` и
  `arcana/bot.py`; при отсутствии ключа бот отвечает «🎤 Голосовые не настроены».
- **Почему круто:** multi-provider (единственный non-Anthropic путь) с
  fail-soft — фича может быть выключена (нет кредитов) без поломки бота. Язык
  захардкожен под домен → точнее распознавание.

## 8. СДВГ-адаптация и женский род — персонализация как системный приём

Пол и нейроотличие — не косметика, а параметры промптов и контракты тестов.
(Содержимое личного профиля здесь намеренно не цитируется — приватность; ниже —
*техника*, а не данные.)

- **Женский род + имя + прямое «ты»** в генеративных промптах:
  `core/memory.py:459` (`_ADHD_TIP_SYSTEM`), `nexus/handlers/tasks.py:1069`
  (`_NUDGE_SYSTEM`), `:2763, :2997`, `miniapp/backend/routes/today.py:177`
  (`_TIP_SYSTEM`).
- **Инжект нейро-профиля**: в `_ADHD_TIP_SYSTEM` / `_NUDGE_SYSTEM` подаётся
  структурированный профиль (паттерны прокрастинации, триггеры) → совет
  *конкретный под пользователя*, а не дженерик «составьте список».
- **СДВГ-aware бизнес-правила в промптах**: бюджетный алгоритм
  (`finance.py:2313`) закладывает анти-импульсные ограничения распределения;
  нудж после создания задачи (`tasks.py:1082`) даёт ОДИН конкретный шаг.
- **Защита «голоса» в whitelist**: спелл-коррекция не трогает термины, на которых
  строится доменный/личный язык (`tests/test_preprocess.py:31-37`).
- **Почему круто:** персона (пол, имя, нейротип) проходит сквозь весь стек —
  парсинг, генерацию и тесты — как явный контракт, а не разовая фраза в одном
  промпте.

## 9. Guardrails — где система не доверяет модели

Самая «инженерная» часть: каждый LLM-выход обёрнут защитой.

### 9.1 Whitelist spell-correction
- **`core/preprocess.py`**: Haiku исправляет опечатки, но в промпт инжектится
  whitelist «НИКОГДА не исправляй»: 78 RU-карт Таро + ~30 эзо-терминов + **имена
  клиентов из Notion**. SQLite-кеш whitelist, `TTL=3600s` (`:37`); при создании
  клиента — `invalidate_whitelist` (`:126`), иначе свежее имя «исправится».
- Реальный баг, который это чинит (`tests/test_preprocess.py:48-60`): Haiku
  превращал имя клиента в нарицательное слово. Whitelist это блокирует.
- **Почему круто:** generic LLM-фича (исправь опечатки) огорожена доменным
  словарём, который **строится из живых данных пользователя** и кешируется с
  инвалидацией.

### 9.2 Валидаторы вывода + retry + fallback (СДВГ-совет)
- **`miniapp/backend/routes/today.py`**: `_TIP_SYSTEM` (`:177`), `_validate_tip`
  (`:163`), вызов Haiku `max_tokens=80, temperature=0.4` (`:195-197`).
  - Контракт: ≤15 слов, 1 предложение, без эмодзи/markdown, без «штука/вещь/
    нечто» (анти-плейсхолдеры).
  - `tests/test_tip_validation.py`: плохой ответ → **retry один раз** → если опять
    плохой → fallback-строка, и **плохой ответ не кешируется** (гигиена кеша).
- **Почему круто:** classic constraint-satisfaction loop вокруг недетерминированной
  модели: validate → retry → fallback, плюс `temperature=0.4` для стабильности
  короткого формата.

### 9.3 Анти-conversational и анти-truncation guard
- **`core/preprocess.py`**: если Haiku вернул «извините, я не могу…» или текст
  слишком длинный/обрезанный — вернуть исходник (`tests/test_preprocess.py:95-110`).
- **`core/classifier.py:732-735`**: если *вход* начинается с «я не могу / как ии /
  i cannot» — вернуть `unknown` (защита от подачи в систему утёкшего ответа LLM).
- **Почему круто:** система защищается от собственных моделей в обе стороны — и от
  «болтливого» спелл-чекера, и от скормленного обратно LLM-ответа.

### 9.4 Adaptive max_tokens
- **Spell-correction**: `max_tokens = max(300, len(converted)//2 + 200)` — длинные
  сессии (5+ триплетов) больше не обрезаются (issue #82).
- **Lists**: `max_tokens=2000` под мульти-айтемные списки (issue #66,
  `tests/test_lists_v1_2.py:145-176` проверяет `>= 2000`).
- **Session parser**: `max_tokens=4000` (`sessions.py:649`).
- **Почему круто:** бюджет токенов масштабируется под длину входа — баланс
  «не платить лишнего» vs «не потерять хвост», закреплён тестами.

### 9.5 JSON-only + эвристический fallback-парсер
- Все Haiku-парсеры: «Ответь ТОЛЬКО валидным JSON без markdown», затем безопасный
  стрип ` ```json ` заборов.
- При невалидном JSON — **эвристический парсер** сохраняет ввод:
  `arcana/handlers/grimoire.py` (`tests/test_grimoire_parse.py:35-73`),
  `arcana/handlers/ritual_writeoff.py`.
- **Почему круто:** структурированный вывод никогда не теряет данные
  пользователя — даже когда модель «сломала» JSON, regex-fallback вытащит
  заголовок/тело.

## 11. Сводка инженерных паттернов

1. **Cheap-by-default** — Haiku как дефолт обёртки, Sonnet требует явного выбора и
   защищён тестом-whitelist'ом.
2. **Regex pre-filter → LLM** — очевидное ловится бесплатно, модель видит только
   неоднозначное (classifier, list-classifier).
3. **Морфология как сигнал** — вид/время русского глагола = planned/done.
4. **Layered prompt assembly** — system-prompt собирается из источников на лету
   (RAG без векторов).
5. **Whitelist guards** — generic LLM-фичи (спелл, vision-категории) огорожены
   доменными словарями из живых данных + кеш с инвалидацией.
6. **Validate → retry → fallback** — недетерминированный вывод всегда обёрнут
   контрактом и деградирует мягко (эвристический парсер, fallback-строка).
7. **Adaptive token budgets** — `max_tokens` под длину входа, закреплён тестами.
8. **Persona-as-contract** — пол/имя/нейротип проходят сквозь парсинг, генерацию
   и тесты.
9. **Prompts as tested artifacts** — содержимое промптов и роутинг моделей под
   регрессионными тестами.
